The frontend needs to be updated to use the backend API instead of mockApi.

EXAMPLE: Update Auth Service

Replace services/authService.ts with API calls:

import axios from 'axios';

const API_URL = 'http://localhost:5000/api';

export const loginUser = async (email: string, password: string, role: string) => {
  const response = await axios.post(`${API_URL}/auth/login`, {
    email, password, role
  });
  
  if (response.data.success) {
    localStorage.setItem('token', response.data.token);
    return { success: true, user: response.data.user };
  }
  return { success: false, message: response.data.message };
};

Repeat this pattern for all services to integrate with backend.

ISSUE: "docker: command not found"
SOLUTION: 
  - Install Docker Desktop
  - Restart terminal after installation
  - Verify: docker --version

ISSUE: "port is already allocated"
SOLUTION:
  - Check what's using the port: netstat -ano | findstr :3306
  - Stop the conflicting service or change port in docker-compose.yml

ISSUE: "Cannot connect to MySQL" in Adminer
SOLUTION:
  - Use server name "mysql" not "localhost"
  - Wait 30 seconds after first startup
  - Check if container is running: docker ps

ISSUE: Backend returns 503 Service Unavailable
SOLUTION:
  - MySQL may not be ready yet, wait 30 seconds
  - Check logs: docker logs attendify_backend
  - Restart: docker restart attendify_backend

ISSUE: Frontend shows blank page
SOLUTION:
  - Check browser console for errors (F12)
  - Ensure backend is running: curl http://localhost:5000/health
  - Clear browser cache (Ctrl+Shift+Delete)

Q: How do I reset the entire database?
A: Run: docker compose down -v (WARNING: Deletes all data)
   Then: docker compose up -d (Re-initializes with seed data)

Q: Can I use this with a remote MySQL server?
A: Yes! Update backend/.env:
   DB_HOST=remote-server-ip
   DB_PORT=3306
   Then comment out mysql service in docker-compose.yml

Q: How do I change default passwords?
A: 
   1. Update docker-compose.yml (MYSQL_PASSWORD, MYSQL_ROOT_PASSWORD)
   2. Update backend/.env (DB_PASSWORD)
   3. Rebuild: docker compose down -v && docker compose up -d

Q: Can multiple users share the same database?
A: Yes, if they connect to the same MySQL server. For local development,
   each PC has its own database. For shared data, set up a central MySQL
   server and have all PCs connect to it.

Q: How do I deploy this to production?
A: 
   1. Set strong passwords in .env files
   2. Use HTTPS (add nginx reverse proxy)
   3. Use managed MySQL (AWS RDS, DigitalOcean)
   4. Build frontend: npm run build
   5. Deploy to hosting (Vercel, Netlify, etc.)

Q: Does this work on mobile?
A: The frontend is mobile-responsive. Access via mobile browser at
   http://<your-pc-ip>:3000. For production, deploy to web hosting.

Q: How do I add more teachers/students/courses?
A: Three ways:
   1. Use app's signup page (users create accounts)
   2. Use admin panel (admin creates accounts)
   3. Use Adminer (insert directly into database)

Q: Can I change the port numbers?
A: Yes! Edit docker-compose.yml:
   ports:
     - "YOUR_PORT:3306"  # MySQL
     - "YOUR_PORT:8080"  # Adminer
     - "YOUR_PORT:5000"  # Backend

Q: How do I backup my data regularly?
A: Set up a cron job (Linux/Mac) or Task Scheduler (Windows):
   
   # Linux/Mac crontab:
   0 2 * * * docker exec attendify_mysql mysqldump -u root -prootpass123 attendify_db > /backup/attendify_$(date +\%Y\%m\%d).sql
   
   # Windows Task Scheduler:
   Run: backup.bat daily at 2 AM
   Content: docker exec attendify_mysql mysqldump -u root -prootpass123 attendify_db > C:\backup\attendify_%date%.sql
